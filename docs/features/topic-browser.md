# Topic Browser

> **Status:** Implemented (2026-01-05)
> **Module:** Intelligence
> **Route:** `/intelligence/topics`
> **Backend:** clustering-service (Port 8122)

## Overview

The Topic Browser provides an interactive interface for exploring semantic topic clusters generated by the clustering-service. It enables users to browse through news topics, understand topic composition through keywords, and drill down into individual articles within each cluster.

## Features

### Topic List View (`/intelligence/topics`)

- **Batch Info Card**: Displays latest clustering batch stats (topic count, article count, timestamp)
- **Topic Grid**: Responsive grid of topic cards showing:
  - Auto-generated topic label (from representative article title)
  - Article count badge
  - Top 5 keywords as tags
- **Search**: Filter topics by keyword (client-side filtering)
- **Min-Size Filter**: Dropdown to filter by minimum article count (3, 5, 10, 20)
- **Pagination**: Load More button for infinite scroll pattern

### Topic Detail View (`/intelligence/topics/:id`)

- **Topic Header**: Label, article count, edit button (planned)
- **Keywords Section**: All extracted keywords as tags
- **Metadata**: Cluster index, batch ID, creation date
- **Sample Articles Table**: Up to 20 representative articles with:
  - Rank number
  - Article title
  - Distance score (similarity to cluster centroid)
  - Publication date
  - External link to source

## API Endpoints

### GET /api/v1/topics

List topic clusters with pagination.

**Query Parameters:**
| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| min_size | int | 10 | Minimum articles per topic |
| limit | int | 50 | Results per page |
| offset | int | 0 | Pagination offset |
| batch_id | string | - | Filter by batch ID |

**Response:**
```json
{
  "topics": [
    {
      "id": 80,
      "cluster_index": 79,
      "label": "Australia crush England...",
      "size": 410,
      "keywords": ["england", "ashes", "australia"],
      "batch_id": "afe6a547-...",
      "created_at": "2026-01-05T20:34:13Z"
    }
  ],
  "total": 578,
  "offset": 0,
  "limit": 50
}
```

### GET /api/v1/topics/:id

Get topic details with sample articles.

**Query Parameters:**
| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| sample_limit | int | 20 | Max sample articles |

**Response:**
```json
{
  "id": 80,
  "cluster_index": 79,
  "label": "Australia crush England...",
  "size": 410,
  "keywords": ["england", "ashes", "australia", "test", "live"],
  "batch_id": "afe6a547-...",
  "created_at": "2026-01-05T20:34:13Z",
  "sample_articles": [
    {
      "article_id": 12345,
      "title": "Australia crush England fightback...",
      "url": "https://...",
      "published_at": "2026-01-05T...",
      "distance": 0.141
    }
  ]
}
```

### GET /api/v1/topics/search

Search topics by keyword.

**Query Parameters:**
| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| q | string | required | Search query |
| limit | int | 20 | Max results |

### GET /api/v1/topics/batches

List clustering batch runs.

**Query Parameters:**
| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| status | string | - | Filter: pending/running/completed/failed |

### POST /api/v1/topics/:id/feedback

Submit feedback to correct topic label.

**Request Body:**
```json
{
  "suggested_label": "Ashes Cricket Series 2026",
  "feedback_type": "label_correction",
  "comment": "More specific label"
}
```

## React Query Hooks

```typescript
const mutation = useSubmitFeedback();

```

## Configuration

### Environment Variables

The Topic Browser uses dynamic hostname detection for remote access. The `.env` variable is optional:

```bash
# Optional - if not set, uses window.location.hostname
# VITE_TOPICS_API_URL=http://localhost:8122/api/v1
```

### CORS Configuration

The clustering-service includes CORS middleware for browser access:

```python
# services/clustering-service/app/main.py
app.add_middleware(
    CORSMiddleware,
    allow_origin_regex=r"https?://(localhost|127\.0\.0\.1|192\.168\.\d+\.\d+)(:\d+)?",
    allow_credentials=True,
    allow_methods=["GET", "POST", "PUT", "DELETE", "OPTIONS"],
    allow_headers=["*"],
)
```

## Usage Examples

### Accessing Topic Browser

1. Navigate to `/intelligence/topics`
2. Browse topics in the grid view
3. Use min-size filter to show only larger clusters
4. Click a topic card to view details and sample articles

### Searching Topics

1. Enter keyword in search box (e.g., "ukraine")
2. Results filter in real-time
3. Clear search to show all topics

### Viewing Topic Details

1. Click any topic card
2. View full keyword list and metadata
3. Browse sample articles with similarity scores
4. Click external link icon to open original article
