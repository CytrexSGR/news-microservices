# Auth Service

The Auth Service provides authentication and authorization for every News MCP component. It exposes JWT-based login flows, role-aware user management, and API key provisioning while maintaining audit trails for compliance.

---

## Highlights

- **JWT access & refresh tokens** with Redis-backed blacklist for logout and compromise handling
- **Role-Based Access Control (RBAC)** including default `user`, `admin`, and `moderator` roles
- **API keys** with prefixing, rotation, and usage tracking for service-to-service access
- **Account protection** via configurable password policy, rate limiting, and lockout counters
- **Audit logging** for authentication events (`auth_audit_log` table)

---

## Tech Stack

| Layer | Detail |

Dependencies are pinned in `requirements.txt`. Development extras live in `requirements-dev.txt`.

## Configuration

All settings derive from environment variables defined in `.env`. Key options (see `app/config.py`):

| Variable | Description | Default |
| --- | --- | --- |
| `DATABASE_URL` | Postgres DSN (`postgresql://...`) | Constructed from `POSTGRES_*` values |
| `REDIS_URL` | Redis DSN for blacklist / rate limiting | Constructed from `REDIS_*` values |
| `JWT_SECRET_KEY` | HS256 signing secret (в‰Ґ32 chars) | Randomly generated if unset |
| `JWT_ACCESS_TOKEN_EXPIRE_MINUTES` | Access token lifetime | 30 minutes |
| `JWT_REFRESH_TOKEN_EXPIRE_DAYS` | Refresh token lifetime | 7 days |
| `RATE_LIMIT_ENABLED` | Toggle request throttling | `true` |
| `RATE_LIMIT_REQUESTS` | Requests allowed per window | 100 |
| `RATE_LIMIT_WINDOW_SECONDS` | Rate limit window size | 3600 |
| `CORS_ORIGINS` | Allowed origins | `["http://localhost:3000", "http://localhost:8080"]` |

When the service starts it:
1. Validates database connectivity (`check_db_connection`).
2. Creates tables if running in development/test (`init_db`).
3. Ensures default roles exist.

## API Overview

All endpoints are namespaced under `/api/v1`.

| Route | Method | Description |
| --- | --- | --- |
| `/auth/register` | POST | Register a new user with password validation |
| `/auth/login` | POST | Issue access + refresh tokens (username or email) |
| `/auth/refresh` | POST | Exchange refresh token for new access token |
| `/auth/logout` | POST | Blacklist current access token |
| `/auth/me` | GET | Retrieve current user profile |
| `/auth/api-keys` | POST/GET/DELETE | Manage API keys tied to the requesting user |
| `/users` | GET | Paginated user list (admin only) |
| `/users/{id}` | GET/PUT/DELETE | Admin/user self-management operations |

Responses use Pydantic schemas from `app/schemas/auth.py`. See autogenerated Swagger docs at `/docs` when running locally.

```bash

```

Tests rely on in-memory SQLite or the configured Postgres instance depending on environment variables. Use `ENVIRONMENT=test` to enable NullPool connections.

## Operational Notes

- **Audit log** entries are created for every login/logout/register attempt via `AuthService.log_auth_event`.
- **Account lockout** increments `failed_login_attempts` and sets `locked_until` after repeated failures (configurable threshold).
- **Service-to-service access**: generate API keys, then present them in downstream services via `Authorization: ApiKey <key>`.
- **Observability**: Logging defaults to JSON and integrates with centralized Loki/Grafana stacks when running under Compose/K8s.

This README reflects the current implementation in the repository (FastAPI v0.115.0 stack). Update it when endpoints or configuration knobs change.

- вњ… **Multi-Key Support:** Service maintains pool of valid JWT signing keys
- вњ… **30-Day Grace Period:** Old tokens remain valid during rotation
- вњ… **Automatic Rotation:** Keys rotate on configurable schedule (default: 30 days)
- вњ… **Zero Downtime:** No service restart required for key updates

**Configuration:**
```env
JWT_KEY_ROTATION_ENABLED=true
JWT_KEY_ROTATION_DAYS=30
JWT_KEY_GRACE_PERIOD_DAYS=30
```

**How It Works:**
1. New key added to pool with current timestamp
2. Both old and new keys accepted for token verification
3. New tokens signed with latest key
4. Old keys expired after grace period
